清网报告模板

发表于 2017-10-19 | 分类于 工作 | 阅读次数

导出数据脚本

# 历史累计总量
# 字段含义依次：恶意总量、拦截总量、告警总量、继续访问总量、停止访问总量
mongoexport -d afui -c stat_count_throughout --csv -f 'total_count,murl_count,block_count,warning_count,go_count,stop_count,area_code' -o stat_count_throughout.csv

# 每日趋势（按类别）
# 字段含义依次：时间点（天）、社工欺诈、信息诈骗、虚假销售、诈骗文件、色情网站、赌博网站
mongoexport -d afui -c trend --csv -f 'fraud_date,fraud_social,fraud_info,fraud_sales,fraud_mware,fraud_porn,fraud_gamble' -o trend.csv

# 每日趋势（按行为）    
# 字段含义依次：时间点（天）、恶意总量、拦截总量、告警总量、继续访问总量、停止访问总量
mongoexport -d afui -c history_stat --csv -f 'day_time,murl_count,block_count,warning_count,stop_count,go_count' -o history_stat.csv

# 每小时趋势（按行为）
# 字段含义依次：时间点（小时）、恶意总量、拦截总量、告警总量、继续访问总量、停止访问总量、地区（这里无意义，忽略）
mongoexport -d afui -c stat_count_hour --csv -f 'hour_time,murl_count,block_count,warning_count,go_count,stop_count,area_code' -o stat_count_hour.csv

# 网址累计访问
# 字段含义依次：行为：拦截/告警、累计访问、是否整域名、恶意分类、URL
mongoexport -d afui -c stat_url --csv -f 'fraud_action,fraud_count,fraud_range,fraud_type,fraud_url' -o stat_url.csv

# 访问网址原始日志
# 字段含义依次：时间点（秒）、恶意类型、URL、IP、是否域名、是否拦截、域名
mongoexport -d afui -c detail_murl --csv -f 'time,etype,url,ip,domain,block,url_host' -o detail_murl.csv

tcpdump 学习笔记

发表于 2017-09-29 | 分类于 shell | 阅读次数

简介

dump the traffic on a network，根据使用者的定义对网络上的数据包进行截获的包分析工具。 tcpdump可以将网络中传送的数据包的“头”完全截获下来提供分析。它支持针对网络层、协议、主机、网络或端口的过滤，并提供and、or、not等逻辑语句来帮助你去掉无用的信息。

Linux 抓包原理

Linux抓包是通过注册一种虚拟的底层网络协议来完成对网络报文(准确的说是网络设备)消息的处理权。当网卡接收到一个网络报文之后，它会遍历系统中所有已经注册的网络协议，例如以太网协议、x25协议处理模块来尝试进行报文的解析处理，这一点和一些文件系统的挂载相似，就是让系统中所有的已经注册的文件系统来进行尝试挂载，如果哪一个认为自己可以处理，那么就完成挂载。

当抓包模块把自己伪装成一个网络协议的时候，系统在收到报文的时候就会给这个伪协议一次机会，让它来对网卡收到的报文进行一次处理，此时该模块就会趁机对报文进行窥探，也就是把这个报文完完整整的复制一份，假装是自己接收到的报文，汇报给抓包模块。
